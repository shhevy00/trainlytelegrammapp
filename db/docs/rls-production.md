# Row Level Security (PostgreSQL)

В приложении изоляция данных тренеров обеспечивается на уровне сервера: каждый запрос выполняется с `trainer_id` из JWT-сессии, без доверия к ID с клиента.

Опционально для дополнительной защиты на уровне БД см. пример политик в [`../rls.example.sql`](../rls.example.sql).

Рекомендации для production:

1. Подключение приложения — отдельный роль с правами только на DML (не superuser).
2. Включить RLS на таблицах с `trainer_id` после проверки политик на staging.
3. Не использовать `service_role` с обходом RLS в обычных API-запросах.

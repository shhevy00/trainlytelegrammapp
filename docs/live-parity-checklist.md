# Live (PostgreSQL) — ручной паритет с mock

Сопоставление сценариев с кодом и журнал прогонов: **[live-parity-notes.md](./live-parity-notes.md)**.

Режим: `.env.local` с `DATABASE_URL`, без `TRAINLY_USE_MOCK_DATA=true`, вход через dev-secret или Telegram.

Прогоните сценарии и отметьте:

1. [ ] Создать клиента (`/clients/new`).
2. [ ] Создать шаблон тренировки для клиента.
3. [ ] Добавить слот расписания.
4. [ ] Старт тренировки с расписания → логгер → завершение с ≥1 заполненным сетом.
5. [ ] Журнал показывает завершённую тренировку; **остаток занятий** уменьшился на 1.
6. [ ] Завершение как **заметка** (`completed_as_note`) — **без** списания занятия.
7. [ ] Добавить оплату занятий (`/add-payment`) — баланс увеличился.
8. [ ] Quick note сохраняется и виден в контексте клиента.
9. [ ] Юридическое согласие: без accept редирект на `/legal-consent`; после accept — доступ к продукту.
10. [ ] При **истёкшем** trial/active (`valid_until` в прошлом) мутации продукта отклоняются, billing доступен.

При расхождении с mock: зафиксируйте шаг и проверьте `lib/server/trainlyMutations.ts` / server actions.
